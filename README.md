
# 🚀 NanoPi R1S-H5 AI 驱动定制固件

### 📖 项目简介

本项目是一次“心血来潮”的尝试。将尘封已久的 **FriendlyElec NanoPi R1S-H5** 重新翻出，在**零编程基础**的情况下，全程由 AI（ChatGPT）驱动完成固件的功能设计、代码优化及编译排错。

目标简单：**榨干 R1S-H5 的每一分性能**，在保持稳定的前提下，打造一个“玩家级性能 + 常用插件”平衡的 OpenWrt/LEDE 固件。

---

### 🛠 定制思路

针对 R1S-H5 的硬件特性，AI 建议并执行了以下优化方案：

* **性能榨干**：深度集成网络加速模块，优化内核参数以降低 CPU 负载。
* **功能全家桶**：预装常用插件（科学上网、广告拦截、内网穿透、Docker 等），到手即用。
* **小白友好**：所有配置逻辑由 AI 审核，即便没有技术背景也能通过此文档了解固件构成。

---

### 📦 核心功能清单

| 分类 | 包含插件/功能 |
| --- | --- |
| **网络加速** | Turbo ACC (流量分载/BBR/DNS加速), IRQ Balance |
| **上网插件** | OpenClash, PassWall 2 |
| **防护/过滤** | AdGuard Home, SmartDNS |
| **穿透/远程** | Tailscale, ZeroTier, DDNS, UPnP |
| **系统/运维** | Docker (含面板), TTYD 终端, 实时流量监测 (NLBWmon) |
| **实用工具** | 网络唤醒 (WOL), 文件传输, SQM 智能队列管理 |

### 🧱 编译说明

* **底座**：基于 OpenWrt / LEDE 源码。
* **开发模式**：`人机协作` —— 本人负责提出需求与执行，AI 负责生成 `config` 修改建议及修复编译报错。
* **现状**：目前处于“先行版”测试阶段，侧重于跑满千兆带宽（视环境而定）与多任务处理的稳定性。
